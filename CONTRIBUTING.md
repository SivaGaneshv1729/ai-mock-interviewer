# 🤝 Contributing to AI Mock Interviewer Pro

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to AI Mock Interviewer Pro. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## 🚀 How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for AI Mock Interviewer Pro. Following these guidelines helps maintainers and the community understand your report.

*   Check the [Issues](https://github.com/SivaGaneshv1729/ai-mock-interviewer/issues) for similar reports.
*   Clearly describe the steps to reproduce the bug.
*   Include information about your environment (OS, Python version, Browser).

### Suggesting Enhancements
*   Clearly describe the enhancement and its benefit.
*   Provide examples of how it would work if possible.

## 🛠️ Local Development Setup

1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/SivaGaneshv1729/ai-mock-interviewer.git
    ```
2.  **Environment Setup**:
    *   Initialize the SQLite database: `python backend/main.py`.
    *   Configure your `.env` keys.
3.  **Branching Strategy**:
    *   Use feature branches (`feature/your-feature`) from `main`.
4.  **Commit Standards**:
    *   Write clear, concise commit messages.
    *   Follow atomic commit patterns.
